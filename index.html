üåê Modern Network Complexity

  • Employee workstations: Laptops and desktops used by staff for daily work.
  • BYOD devices: "Bring Your Own Device"‚Äîpersonal phones or tablets connected to the company network.

Extended Environment

  • Remote offices: Branches in different locations connected via the internet.
  • Cloud applications: Services like Google Drive or Microsoft 365 used for storing and sharing files online.

  • Third-party providers: External companies that help with IT support or cloud hosting.

üõ°Ô∏è Security Architecture Models

Defense-in-Depth

  • Multiple security layers: Like having a fence, a locked door, and an alarm system at home. If one fails, others still protect you.
  • Each layer counters specific threats: For example, antivirus stops malware, while firewalls block unwanted network traffic.
  • Redundant security controls: If one control is bypassed, another is still in place.

Zero Trust

  • "Never trust, always verify": Every user and device must prove who they are, every time.
  • "Assume breach" mentality: Act as if attackers are already inside, so always check and monitor.
  • Continuous verification: Keep checking users and devices, not just at login.
  • Least privilege access: Give people only the access they need, nothing more.

üîí Defense-in-Depth Architecture

Key Components (with cybersecurity & everyday examples):

  • Policies & Procedures: Like house rules‚Äî"lock the door when you leave" (home) or "use strong passwords" (work).
  • Access Restrictions: Only parents can open the safe at home; only IT staff can install software at work.
  • Network Security: A fence around your house (home) or a firewall blocking bad websites (work).
  • Device Protection: Antivirus is like a security dog that barks at intruders; only approved apps can run, like only letting friends in your house.
  • Application & Data Protection: Backing up photos to the cloud (home) or encrypting company files (work).
Example: Security layers (Unsplash)

Effectiveness: Like having several locks on your door‚Äîif one fails, others still protect you.

üö´ Zero Trust Model

Core Principles (explained):

  • Never Trust, Always Verify: Like checking a delivery person's ID before letting them in, even if they're wearing a uniform. In cybersecurity, always require passwords and extra checks (multi-factor authentication).
  • Assume Breach: At home, you might keep valuables hidden even if you trust your guests. In cybersecurity, always monitor for suspicious activity, just in case.
  • Deny-by-default: Like keeping your room locked unless someone asks for permission. In IT, block access unless a user is approved.

Implementation Features (examples):

  • Monitoring: Like a security camera at home, software watches for strange logins or file access.
  • Logging: Keeping a diary of who entered your house; in IT, logs show who did what and when.
  • Automated detection: Like a burglar alarm, alerts if someone tries to break in or steal data.
  • Incident response: Having a plan to call the police or lock doors quickly; in IT, quickly lock accounts or disconnect devices if something suspicious happens.
Example: Zero Trust (Unsplash)

‚ö†Ô∏è Risk Assessment Framework

Key Steps in Risk Assessment (with examples):

  1. Identify Threat Sources: Who or what could cause harm? (e.g., hackers, storms, careless employees)
  2. Identify Threat Events: What could go wrong? (e.g., a hacker tries to steal data, a flood damages computers)
  3. Identify Vulnerabilities: What weaknesses exist? (e.g., weak passwords, unlocked doors)
  4. Estimate Probability: How likely is it? (e.g., "We get storms every year" or "We never change our passwords")
  5. Assess Adverse Effects: What could happen? (e.g., loss of money, data, or reputation)
  6. Determine Risk Level: Combine likelihood and impact (e.g., "If a hacker gets in, we could lose all our customer data")

Practical implementation of cybersecurity controls

üìä 1. Inventory of Hardware and Software Assets

Risks Without Asset Inventory:

  • Like not knowing how many doors or windows your house has‚Äîhard to secure what you don't know about.
  • Attackers look for forgotten computers or devices to break into.
  • Unregistered machines may have weak security settings.
  • Unknown devices (like a visitor's phone) can connect and cause problems.
  • If something goes wrong, it's hard to investigate if you don't know what should be there.
Example: Asset inventory (Unsplash)

‚öôÔ∏è 2. Secure Configuration

Configuration Risks (with examples):

  • Default Settings: Like leaving your luggage unlocked at the airport. In IT, using "admin/admin" as a password is very risky.
  • Vulnerable Protocols: Like using an old, rusty lock. Outdated software or network protocols are easier to break into.
  • Poor Patch Management: Not fixing a broken window at home. In IT, not updating software leaves known holes open for attackers.
  • Unauthorized Changes: Someone changes your house locks without telling you. In IT, unauthorized changes can weaken security.
Example: Configuration risk (Unsplash)

üõ°Ô∏è 3. Application and Services Execution Control

Attack Surface Concept:

  • More applications = Larger attack surface
  • Multiple access points for attackers
  • More data exposed to risks

Modern Threats:

  • Fileless Attacks: Run in memory, avoid detection
  • Phishing: Malicious attachments and links
  • PowerShell Abuse: Using legitimate tools maliciously

Principle: Apply least functionality - run only necessary applications

üîê 4. Access Control

Access Control Risks:

  • Unnecessary administrator privileges
  • Excessive access rights for roles
  • Malware execution with elevated permissions
  • Lateral movement within networks

Key Principles:

  • Least Privilege: Minimum necessary access
  • Need-to-Know: Access based on job requirements
  • Role-Based Control: Defined roles and permissions
  • Account Separation: Separate admin and user accounts
