üõ°Ô∏è Security Architecture Models

Defense-in-Depth

  • Multiple security layers
  • Each layer counters specific threats
  • Layered protection approach
  • Redundant security controls

Zero Trust

  • "Never trust, always verify"
  • "Assume breach" mentality
  • Continuous verification
  • Least privilege access

üîí Defense-in-Depth Architecture

Key Components:

  • Policies & Procedures: Risk analysis, user training, supply chain management
  • Access Restrictions: Least privilege, need-to-know principles
  • Network Security: Segmentation, firewalls, IDS, VPNs
  • Device Protection: Antivirus, application whitelisting
  • Application & Data Protection: Patching, backup, encryption

Effectiveness: Reduces chance of successful attacks and minimizes damage

üö´ Zero Trust Model

Core Principles:

  • Never Trust, Always Verify
    • Every user/device is untrustworthy
    • Explicit authentication required
    • Minimum required privileges
  • Assume Breach
    • Devices may be compromised
    • Deny-by-default principle
    • Multi-parameter verification

Implementation Features:

  • Detailed monitoring procedures
  • Comprehensive logging
  • Automated suspicious activity detection
  • Immediate incident response

‚ö†Ô∏è Risk Assessment Framework

Key Steps in Risk Assessment:

  1. Identify Threat Sources: Malicious groups, competitors, states, physical threats, errors
  2. Identify Threat Events: Cyber attacks, natural disasters, material damage
  3. Identify Vulnerabilities: System weaknesses that can be exploited
  4. Estimate Probability: Likelihood of threats and successful completion
  5. Assess Adverse Effects: Impact on functions, individuals, organizations, national security
  6. Determine Risk Level: Probability √ó Impact calculation

üìä 1. Inventory of Hardware and Software Assets

Risks Without Asset Inventory:

  • Cannot track legally established resources
  • Attackers scan for unprotected systems
  • Unregistered machines with insecure configurations
  • Unidentified mobile device connections
  • Difficult incident response and forensics

Develop policies, maintain accurate inventory, assign owners

Classify by criticality, control portable media usage

1.6-1.8: BYOD Management

Secure bring-your-own-device policies and configurations

1.9-1.10: Automated Discovery

Regular network scanning and automated inventory updates

‚öôÔ∏è 2. Secure Configuration

Configuration Risks:

  • Default Settings: Admin/admin credentials, unnecessary privileges
  • Vulnerable Protocols: Older, insecure protocol versions
  • Poor Patch Management: Known vulnerabilities remain unpatched
  • Unauthorized Changes: Security configurations modified without authorization

üîß Secure Configuration - Key Controls

2.1-2.3: Basic Security

Policies, remove unsupported systems, change default credentials

2.4-2.8: Updates & Patches

Secure configurations, supported versions, automated patching

2.9-2.11: Network & Access

Host firewalls, network device security, account management

2.12-2.15: Advanced Controls

Change management, media restrictions, service accounts, system images

üõ°Ô∏è 3. Application and Services Execution Control

Attack Surface Concept:

  • More applications = Larger attack surface
  • Multiple access points for attackers
  • More data exposed to risks

Modern Threats:

  • Fileless Attacks: Run in memory, avoid detection
  • Phishing: Malicious attachments and links
  • PowerShell Abuse: Using legitimate tools maliciously

Principle: Apply least functionality - run only necessary applications

üîê 4. Access Control

Access Control Risks:

  • Unnecessary administrator privileges
  • Excessive access rights for roles
  • Malware execution with elevated permissions
  • Lateral movement within networks

Key Principles:

  • Least Privilege: Minimum necessary access
  • Need-to-Know: Access based on job requirements
  • Role-Based Control: Defined roles and permissions
  • Account Separation: Separate admin and user accounts
